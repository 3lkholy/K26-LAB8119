# K26-LAB8119

## Build an AI Powered Case Triage Agent for Financial Services

Financial Services teams face intense regulatory pressure to handle customer complaints quickly, consistently, and with full auditability. Complaints often arrive as unstructured inbound email, yet require precise interpretation and defensible action.

In this hands on lab, you will build an agentic AI powered complaint triage workflow. You will configure AI agents that classify complaint intent, research relevant context, and guide supervised case creation and routing. By the end of the lab, you will have a working agentic workflow and a practical blueprint for compliant complaint handling at scale.

## Repository contents (preloaded lab assets)

This repository contains assets that are preconfigured and preloaded for the lab environment. These artifacts support the exercises but are not created during the lab unless explicitly stated in the lab guide.

### Agents
This directory contains prebuilt AI agents that classify customer interactions and support outbound email communication. These agents are referenced during the lab to demonstrate agentic complaint triage patterns.

✅ Upload the agents before executing the lab.

### Interactions
This directory contains Interaction records that simulate incoming customer emails. These records create Interaction entries in ServiceNow and are used to test and validate the complaint triage workflow during the hands on exercises.

✅ Upload these records to your ServiceNow instance before starting the lab.

### Knowledge Base
This directory contains Knowledge Base articles used by the agents to research context and provide guided responses during the lab.

✅ Upload these articles to your ServiceNow Knowledge Base for use in the exercises.
